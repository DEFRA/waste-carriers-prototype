Prototype change log

This summarises the main changes we've made in response to feedback.

Version 1 to 2

Version 2

Summary of change in v2 User need, reason or source
Receipt email to be specified within journey to ensure registration contact and payee can have different email addresses. So that if an agent applied on behalf of a carrier the carrier doesn't phone EA to query the price difference once they get a receipt, additionally, so that any user in general completing the form can have the receipt sent to another party without having to forward the details themselves. If the receipt email is left blank it'll instead go to the contact email as before. https://eaflood.atlassian.net/browse/RUBY-1126
Final reminder flow to include postal return for updating email address and resending magic link, in addition to an online route with renewal code and short url. So that if a user does not renew after two renewal emails they can receive another reminder, this time offline, in case their details are out of date or the email is missed. This version does not rely on typing the magic link which proved very unpopular and barely usable as a url to type manually. https://eaflood.atlassian.net/browse/RUBY-1126
